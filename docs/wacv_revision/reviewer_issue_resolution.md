# Reviewer Issue Resolution

## 1. 创新性不足

### 解决方式

- 明确将论文定位为“可靠性建模”而非“更强 backbone”。
- 追加置信度信号有效性证据，证明收益不能由“多一个输入通道”解释。

### 现有证据

- 在同一代码路径下，`original / constant / shuffled / removed confidence` 的四流 Top-1 分别为：
  - `52.35% / 46.14% / 47.32% / 41.78%`
- 结论：原始置信度明显优于常数置信度、打乱置信度和完全去除置信度，说明收益来自“置信度与观测质量的对应关系”，而不是任意附加一个数值通道。

### 建议答辩句

本文的创新点不在于“使用 confidence”本身，而在于将其从姿态估计阶段的附带输出提升为贯穿输入编码、传播、修复、融合和训练目标的统一可靠性先验，并在统一主干下给出可归因的鲁棒性证据。

## 2. 合成扰动真实性不足

### 解决方式

- 将合成扰动明确定位为“可复现、可归因的分析协议”。
- 同时补充低质量样本结果和公开强基线比较，避免证据完全依赖 synthetic corruption。

### 现有证据

- 低质量样本 Top-1：
  - baseline：`58.29%`
  - struct：`59.30%`
  - consistency：`63.32%`
- 与公开 DSTA-SLR 强基线在相同扰动协议下比较：
  - clean：`-0.84`
  - missing 10%：`+8.89`
  - missing 20%：`+27.68`
  - missing 30%：`+34.06`
  - noise 20：`-10.23`

### 建议答辩句

本文并未将合成扰动结果解释为对真实部署退化分布的完整替代，而是将其作为可复现、可归因的分析协议；同时，我们也补充了低质量样本结果和公开强基线对比，以支持该方法在真实质量退化相关场景下的价值判断。

## 3. clean-set 不是最优

### 解决方式

- 不再把 clean-set 排名作为核心 claim。
- 将 clean-set 结果稳定表述为“保持竞争性性能”。

### 现有证据

- WLASL100：`78.86 / 79.59`，低于 DSTA-SLR `82.38 / 83.09`
- 公开 DSTA-SLR joint-stream clean：`66.44`
- 本文 struct clean：`65.60`

### 建议答辩句

本文的价值不在于提出新的最强骨架主干，而在于证明可靠性建模本身是一个值得独立研究的设计维度；它关注的是退化观测下的稳定性，而不是继续提升 clean-set 上限。

## 4. confidence 未校准

### 解决方式

- 在方法与实验设置中明确：confidence 仅做清洗与裁剪，不做额外校准。
- 在论文中反复强调：confidence 被用作相对可靠性排序信号，而非校准概率。

### 建议答辩句

本文并不将 confidence 解释为校准后的正确率概率，而仅将其作为相对可靠性排序信号使用。框架需要的是高低关系，而不是精确概率；这也是我们在局限性中明确提出未来需进一步做跨估计器和显式校准验证的原因。

## 5. noise 20 下结构增强低于基线

### 解决方式

- 不回避该现象，而是把它解释为模块功能分工的直接证据。
- 将结论从“结构模块全面提升鲁棒性”改为“结构模块主攻缺失，一致性训练补足强噪声”。

### 现有证据

- baseline `noise 20`：`51.34%`
- struct `noise 20`：`44.80%`
- consistency `noise 20`：`58.56%`

### 建议答辩句

这一现象并不否定本文结论，反而说明结构可靠性建模与一致性训练在不同退化类型下承担不同角色：结构模块主要提升缺失型退化稳定性，而强噪声场景下的收益主要来自训练阶段的一致性约束。

## 6. Table 4 与 Table 5/8 口径不同

### 解决方式

- 在正文中明确三张表分别服务于不同问题。
- 避免读者把 clean 条件下的模块分析与鲁棒性协议下的结果直接横向比较。

### 建议答辩句

Table 4 主要用于分析 clean 条件下单流与四流的模块作用，Table 5/8 主要用于分析统一受控扰动协议下的结构收益与训练收益。两者分别回答“模块是否有效”和“谁在退化条件下贡献了鲁棒性收益”，因此口径不同但不存在数字冲突。

## 最后一条总口径

本文最重要的贡献不是提出一个新的最强骨架主干，而是证明了：在退化骨架观测下，显式可靠性建模是一条低开销、可插拔且具有明确归因证据的有效路径。
