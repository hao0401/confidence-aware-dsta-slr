# 局限性与答辩口径

## 论文中建议使用的“局限性与边界”版本

尽管本文结果在统一受控扰动协议下支持所提方法的鲁棒性优势，但仍需明确其适用边界。首先，当前关键点缺失和高斯噪声实验属于可控分析，其目的在于建立可复现、可归因的相对比较，而非完整刻画真实部署中的退化分布；因此，本文结论不能直接外推到长时遮挡、系统性偏移、多主体交互或高置信误检等更复杂场景。其次，本文依赖姿态估计器输出的置信度通道，且当前仅对该通道进行清洗与裁剪，而未进行额外校准，因此方法收益在多大程度上受置信度质量与姿态估计器类型影响，仍需跨估计器验证。再次，本文的核心证据建立在统一主干、统一预处理与统一协议下的相对比较之上，其意义在于证明可靠性建模是骨架孤立词手语识别中一个有效且可单独研究的设计维度，而非声称已经解决该任务中的全部鲁棒性问题。未来工作将进一步结合真实低质量样本、跨姿态估计器设置、重复实验统计以及更显式的不确定性校准，对该框架在复杂部署条件下的适用性进行更系统的评估。

## 答辩总口径

### 1. 创新性不足，只是把 confidence 拿来加权

短答：

本文的创新点不在于“使用 confidence”本身，而在于将其从姿态估计阶段的附带输出提升为贯穿输入编码、图传播、时间修复、融合和训练目标的统一可靠性先验，并在统一主干下给出可归因的鲁棒性证据。

展开答法：

- 不是单点加权或附加通道，而是贯穿整条识别链路的统一建模。
- 研究目标也不同：本文关注退化观测下的稳定性，而不是继续提升 clean-set 排名。
- 实验中通过统一主干、统一预处理和统一扰动协议，尽量将收益归因到可靠性建模本身。

### 2. 主要实验基于合成扰动，真实性不足

短答：

本文将受控扰动实验定位为“可归因的分析协议”，而不是对真实部署退化分布的完整替代。

展开答法：

- 合成缺失和高斯噪声的作用是建立稳定、可复现、可比较的鲁棒性基准。
- 正因真实退化分布复杂，才更需要统一协议来隔离方法因素。
- 论文已明确该结论不能直接外推到全部真实场景，并在局限性中注明这一边界。

### 3. clean-set 精度不是最优，为什么这篇文章仍有价值

短答：

本文的价值不在于提出新的最强骨架主干，而在于证明可靠性建模本身是一个值得独立研究的设计维度。

展开答法：

- 许多工作关注 clean-set 上限，本文关注 degraded-input 下限。
- 在真实部署中，稳定性往往比少量 clean-set 排名差异更有应用价值。
- 方法额外开销极低，因此更适合作为现有强主干的插件式增强模块。

### 4. confidence 没有校准，为什么可以直接用

短答：

本文并不将 confidence 解释为校准后的正确率概率，而只将其视为相对可靠性排序信号。

展开答法：

- 进入模型前仅做清洗与裁剪，不假设其为严格概率。
- 框架需要的是“高低相对关系”，不是精确置信概率。
- 这也是本文在局限性中强调未来需进一步做跨估计器和显式校准验证的原因。

### 5. 为什么结构增强版本在强噪声下会低于基线

短答：

这并不否定本文结论，反而说明结构可靠性建模与一致性训练在不同退化类型下承担不同角色。

展开答法：

- 结构模块主要改善缺失型退化，因为其核心作用是抑制不可靠传播并修复局部失真。
- 强噪声场景下，仅靠结构先验并不足够，需要训练阶段显式约束稳定性。
- 表 5 和表 8 的结果共同说明：一致性训练正是在这一场景下补足结构模块的不足。

### 6. Table 4 与 Table 5/8 的数字口径看起来不一样

短答：

这三张表服务于不同问题，因此口径不同，但并不存在正文与表格的数值冲突。

展开答法：

- Table 4 主要分析 clean 条件下单流与四流的模块作用。
- Table 5/8 主要分析统一受控扰动协议下的结构收益与训练收益。
- 二者分别回答“模块是否有效”和“在退化条件下谁贡献了鲁棒性收益”，因而不能直接横向比较绝对数值。

## 建议避免的答辩说法

- 不要说“我们全面优于现有方法”。
- 不要说“confidence 本身就是概率，因此直接可用”。
- 不要说“合成扰动已经足够代表真实场景”。
- 不要回避 `noise 20` 下结构增强低于基线的现象，应主动解释其功能分工。

## 建议的最后一句

本文最重要的贡献不是提出一个新的最强骨架主干，而是证明了：在退化骨架观测下，显式可靠性建模是一条低开销、可插拔且具有明确归因证据的有效路径。
