# 补充材料

## S1. 置信度信号有效性分析

本文的核心假设之一是：关键点置信度并非必须被解释为校准后的正确率概率，但其高低顺序仍可作为有用的相对可靠性信号。为验证本文收益不是由“额外增加一个输入通道”这一简单因素带来，本文在同一代码路径下比较四种置信度设置：`original confidence`、`constant confidence`、`shuffled confidence` 和 `removed confidence signal`。其中，`original confidence` 使用预处理骨架文件原始提供的关键点置信度；`constant confidence` 将全部关键点置信度替换为常数；`shuffled confidence` 保持数值分布不变但打乱其与时空位置的对应关系；`removed confidence signal` 则完全去除该通道。

该实验的目的不是进一步追求绝对最优结果，而是回答一个更直接的问题：本文收益是否真的依赖于“置信度与观测质量之间的对应关系”。若仅仅增加一个额外通道即可获得类似收益，则本文关于“可靠性建模”的论证将明显削弱；相反，若只有原始置信度能够带来最强结果，则说明该信号确实携带了对识别有用的质量信息。

表 S1 给出了 WLASL100 上的结果。可以看到，四流 Top-1 从高到低依次为 `original > shuffled > constant > removed`，且原始置信度明显优于其余设置。这说明：1）模型收益不能简单归因于增加一个额外输入通道；2）保留原始置信度与观测质量之间的时空对应关系是重要的；3）本文将置信度用作可靠性先验是有经验依据的，而不仅仅是直觉性设计。

**表 S1 置信度信号有效性分析（WLASL100）**

| 置信度设置 | 关节点单流 Top-1 (%) | 关节点单流 P-C (%) | 四流 Top-1 (%) | 四流 P-C (%) |
| --- | ---: | ---: | ---: | ---: |
| Original confidence | 34.06 | 35.07 | 52.35 | 53.55 |
| Constant confidence | 30.70 | 31.70 | 46.14 | 47.18 |
| Shuffled confidence | 28.69 | 28.97 | 47.32 | 48.50 |
| Removed confidence signal | 25.50 | 26.51 | 41.78 | 42.62 |

综合表 S1，本文更合理的表述不是“置信度本身就是概率，因此可以直接使用”，而是“原始置信度中保留了与观测质量相关的相对信息，这些信息能够被统一可靠性建模框架有效利用”。这也解释了为什么本文在正文中始终将置信度定义为**相对可靠性排序信号**，而非校准后的概率。

## S2. 与公开强基线在统一扰动协议下的比较

主文中的统一主干对比主要用于隔离结构模块与训练策略的贡献，但审稿人仍可能追问：与公开强方法相比，在相同扰动协议下本文方法究竟处于什么位置。为此，本文进一步选取公开可得的 DSTA-SLR 强基线 checkpoint，并在相同 missing/noise 协议下进行 joint-stream 对比。需要强调的是，该实验的目标不是声称本文方法“全面优于公开强基线”，而是更清楚地刻画其优势边界。

表 S2 给出了比较结果。可以看到，本文结构增强版本在 clean 条件和强噪声条件下并不占优；但在关键点缺失场景下优势明显，且缺失比例越高优势越大。例如，在 `missing 20%` 和 `missing 30%` 条件下，本文方法分别相对公开 DSTA-SLR 强基线提升 `+27.68` 和 `+34.06` 个百分点。这一结果与主文 Table 5/8 的观察一致，即结构可靠性建模主要针对缺失型退化，而强噪声场景仍需一致性训练进一步补足。

**表 S2 与公开 DSTA-SLR 强基线在统一扰动协议下的 joint-stream 比较**

| 场景 | DSTA-SLR Top-1 (%) | Ours Top-1 (%) | 差值 |
| --- | ---: | ---: | ---: |
| Clean | 66.44 | 65.60 | -0.84 |
| Missing 10% | 54.53 | 63.42 | +8.89 |
| Missing 20% | 30.37 | 58.05 | +27.68 |
| Missing 30% | 14.77 | 48.83 | +34.06 |
| Noise 5 | 64.77 | 63.76 | -1.01 |
| Noise 10 | 62.42 | 59.06 | -3.36 |
| Noise 20 | 53.69 | 43.46 | -10.23 |

表 S2 的意义在于帮助读者更准确地理解本文定位。它说明本文方法不是一个在所有条件下都优于现有强方法的新 backbone，而是一个更擅长处理**缺失型退化**的鲁棒性增强模块。对于真实部署而言，这种定位比单纯追求 clean-set 排名更具解释力，也与本文主文的研究目标保持一致。

## S3. 主文 Table 4 与 Table 5/8 的口径说明

主文 Table 4、Table 5 和 Table 8 关注的问题并不相同，因此其结果不应被直接作同口径横向比较。Table 4 的主要目的，是分析不同结构模块在 clean 条件下对单流表征和多流融合的影响，回答“模块是否有效、主要作用于链路的哪个环节”；而 Table 5 和 Table 8 则进一步回答“在统一受控扰动协议下，结构增强与训练约束各自贡献了什么鲁棒性收益”。换言之，前者偏重**模块作用分析**，后者偏重**退化鲁棒性归因**。

这一口径差异也解释了为何 Table 4 中“仅时间校正”在关节点单流上达到最高值，而 Table 5/8 中完整模型与一致性增强版本在整体鲁棒性上更具优势。二者并不矛盾，因为单流 clean 条件下的局部优势并不必然对应多流退化条件下的整体优势。本文关心的不是要求所有指标都由同一配置在所有条件下取得绝对最优，而是通过统一协议刻画：不同结构环节如何围绕可靠性先验协同工作，以及它们在不同退化类型下承担怎样的功能分工。

## S4. 关于 `noise 20` 现象的补充说明

主文中一个容易被注意到的现象是：结构增强版本在 `noise 20` 条件下低于基线（`44.80% < 51.34%`）。本文并不回避这一结果，而将其视为结构可靠性建模与一致性训练之间功能分工的直接证据。具体而言，结构模块主要通过输入编码、传播约束和时间修复抑制不可靠观测的扩散，因此更适合应对关键点缺失、局部失真和短时不连续等退化；而在强像素级噪声条件下，仅靠结构先验不足以保证表征稳定，此时训练阶段的一致性约束更加关键。也正因如此，一致性增强版本在 `noise 20` 条件下进一步提升到 `58.56%`。

因此，`noise 20` 下结构增强版本低于基线，并不否定本文结论，反而与主文 Table 5 和 Table 8 一起表明：**结构可靠性建模主要改善缺失型退化，而一致性训练负责补足强噪声场景。** 这一功能分工使本文方法的定位更清晰，也使其整体证据链更完整。

## S5. 补充结论

上述补充实验进一步强化了主文的两个核心结论。其一，本文收益不能被解释为“增加一个额外通道”或“简单加权”的结果，因为原始置信度与打乱/常数/移除置信度之间存在清晰性能差距。其二，本文方法的优势具有明确边界：它在缺失型退化场景下优势显著，但在 clean 条件和强噪声条件下并非全面占优。这两点共同支撑了本文对自身工作的最终定位，即：**本文的贡献不是提出新的最强骨架主干，而是证明了显式可靠性建模是一条低开销、可插拔且具有明确归因证据的有效路径。**
